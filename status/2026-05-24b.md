## Code additions

- `data/strains.ts` — 11-strain seed converted from the SEO matrix doc;
  every `/strains/[slug]` page prerenders from this until Neon lands.
- `lib/push.ts` — real Web Push fan-out using `web-push` (added as dep).
  Reads VAPID env, signs payload, hits each subscription. Fail-open if
  env vars missing.
- `components/push-subscribe-button.tsx` — drop-in client component
  that registers /service-worker.js + subscribes via PushManager.
- `public/service-worker.js` — handles `push` + `notificationclick`
  events, focuses existing tabs when possible, no external libs.
- `app/sitemap.ts` + `app/robots.ts` — Next metadata routes that serve
  XML/text. robots disallows everything until Brendon flips launch.
- `scripts/openclaw_watcher.py` — extended to handle the `qr-sheets/`
  bucket: pairs `*_sheet.{png,jpg,pdf}` with `*_tokens.txt`, POSTs to
  `/api/admin/qr-sheets`. Stdlib-only (no pyzbar dep — relies on the
  explicit token file). Honors the same VAPID/auth headers as the asset
  ingest path.
- `app/api/notifications/new-product/route.ts` — now actually calls
  `lib/push.broadcast()` instead of returning `queued: 0`.

## Handoff doc added

- `handoff/CLIENT_DISPENSARY_DB_DESIGN.md` — answers Brendon's "come up
